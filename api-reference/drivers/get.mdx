---
title: "GET /drivers/{id} — Chauffeursprofiel ophalen op uniek ID"
sidebarTitle: "Ophalen op ID"
description: "Haal de volledige profielgegevens en huidige beschikbaarheidsstatus op van één specifieke chauffeur via het unieke chauffeur-ID in Transportklok."
---

Met dit endpoint haalt u het volledige profiel op van één chauffeur aan de hand van diens unieke identifier. Het antwoord bevat alle persoonsgegevens, rijbewijsklassen en de huidige beschikbaarheidsstatus, zodat u altijd een actueel beeld heeft van de betreffende chauffeur.

## Endpoint

```
GET https://api.transportklok.nl/v1/drivers/{id}
```

## Authenticatie

Voeg de volgende header toe aan elk verzoek:

```
Authorization: Bearer <api_key>
```

## Padparameters

<ParamField path="id" type="string" required>
  De unieke identifier van de chauffeur. Begint altijd met het voorvoegsel `drv_`. Voorbeeld: `drv_01HX4K2QZ9P8V8`.
</ParamField>

## Voorbeeldverzoek

```bash
curl --request GET \
  --url "https://api.transportklok.nl/v1/drivers/drv_01HX4K2QZ9P8V8" \
  --header "Authorization: Bearer <api_key>" \
  --header "Content-Type: application/json"
```

## Voorbeeldrespons (200 OK)

```json
{
  "data": {
    "id": "drv_01HX4K2QZ9P8V8",
    "name": "Jan de Vries",
    "email": "jan.devries@example.com",
    "phone": "+31612345678",
    "license_classes": ["B", "C", "CE"],
    "status": "available",
    "created_at": "2024-01-15T09:00:00Z",
    "updated_at": "2024-06-10T11:00:00Z"
  }
}
```

## Responsvelden

<ResponseField name="data" type="object">
  Het chauffeursobject voor de opgegeven identifier.

  <Expandable title="Chauffeurobject">
    <ResponseField name="data.id" type="string">
      De unieke identifier van de chauffeur, voorafgegaan door `drv_`.
    </ResponseField>

    <ResponseField name="data.name" type="string">
      De volledige naam van de chauffeur.
    </ResponseField>

    <ResponseField name="data.email" type="string">
      Het e-mailadres van de chauffeur.
    </ResponseField>

    <ResponseField name="data.phone" type="string">
      Het telefoonnummer van de chauffeur in internationaal formaat (E.164).
    </ResponseField>

    <ResponseField name="data.license_classes" type="array">
      Een array van rijbewijsklassen die de chauffeur bezit. Mogelijke waarden: `B`, `C`, `CE`.
    </ResponseField>

    <ResponseField name="data.status" type="string">
      De huidige beschikbaarheidsstatus van de chauffeur. Mogelijke waarden: `available`, `on_trip`, `unavailable`, `inactive`.
    </ResponseField>

    <ResponseField name="data.created_at" type="string">
      Tijdstip waarop het chauffeursprofiel is aangemaakt, in ISO 8601-formaat.
    </ResponseField>

    <ResponseField name="data.updated_at" type="string">
      Tijdstip van de laatste wijziging aan het chauffeursprofiel, in ISO 8601-formaat.
    </ResponseField>
  </Expandable>
</ResponseField>

## Foutrespons — chauffeur niet gevonden

Wanneer er geen chauffeur bestaat met het opgegeven ID, retourneert de API een `404 Not Found`-respons.

```json
{
  "error": {
    "code": "not_found",
    "message": "No driver found with ID drv_01HX4K2QZ9P8V8."
  }
}
```

<Warning>
  Zorg ervoor dat u het volledige chauffeur-ID inclusief het voorvoegsel `drv_` meestuurt in het pad. Een onvolledig of onjuist ID leidt altijd tot een `404`-fout.
</Warning>
