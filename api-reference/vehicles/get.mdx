---

Met dit endpoint haalt u de volledige registratie op van één voertuig aan de hand van diens unieke identifier. Het antwoord bevat alle technische specificaties, de APK-vervaldatum en de huidige beschikbaarheidsstatus, zodat u altijd over actuele voertuiginformatie beschikt bij het plannen van transporten.

## Endpoint

```
GET https://api.transportklok.nl/v1/vehicles/{id}
```

## Authenticatie

Voeg de volgende header toe aan elk verzoek:

```
Authorization: Bearer <api_key>
```

## Padparameters

<ParamField path="id" type="string" required>
  De unieke identifier van het voertuig. Begint altijd met het voorvoegsel `veh_`. Voorbeeld: `veh_01HX4K2QZ9P8V9`.
</ParamField>

## Voorbeeldverzoek

```bash
curl --request GET \
  --url "https://api.transportklok.nl/v1/vehicles/veh_01HX4K2QZ9P8V9" \
  --header "Authorization: Bearer <api_key>" \
  --header "Content-Type: application/json"
```

## Voorbeeldrespons (200 OK)

```json
{
  "data": {
    "id": "veh_01HX4K2QZ9P8V9",
    "license_plate": "12-AB-34",
    "brand": "DAF",
    "model": "XF 530",
    "type": "truck",
    "payload_kg": 24000,
    "volume_m3": 82,
    "status": "available",
    "apk_expiry": "2025-03-15",
    "created_at": "2024-01-10T08:00:00Z"
  }
}
```

## Responsvelden

<ResponseField name="data" type="object">
  Het voertuigobject voor de opgegeven identifier.

  <Expandable title="Voertuigobject">
    <ResponseField name="data.id" type="string">
      De unieke identifier van het voertuig, voorafgegaan door `veh_`.
    </ResponseField>

    <ResponseField name="data.license_plate" type="string">
      Het kenteken van het voertuig in Nederlands formaat.
    </ResponseField>

    <ResponseField name="data.brand" type="string">
      Het merk van het voertuig.
    </ResponseField>

    <ResponseField name="data.model" type="string">
      Het model van het voertuig.
    </ResponseField>

    <ResponseField name="data.type" type="string">
      Het type voertuig. Mogelijke waarden: `van`, `truck`, `semi_trailer`.
    </ResponseField>

    <ResponseField name="data.payload_kg" type="integer">
      De maximale laadcapaciteit van het voertuig in kilogram.
    </ResponseField>

    <ResponseField name="data.volume_m3" type="integer">
      Het laadvolume van het voertuig in kubieke meter.
    </ResponseField>

    <ResponseField name="data.status" type="string">
      De huidige beschikbaarheidsstatus van het voertuig. Mogelijke waarden: `available`, `in_use`, `maintenance`, `inactive`.
    </ResponseField>

    <ResponseField name="data.apk_expiry" type="string">
      De vervaldatum van de APK-keuring in `YYYY-MM-DD`-formaat.
    </ResponseField>

    <ResponseField name="data.created_at" type="string">
      Tijdstip waarop het voertuig is geregistreerd, in ISO 8601-formaat.
    </ResponseField>
  </Expandable>
</ResponseField>

## Foutrespons — voertuig niet gevonden

Wanneer er geen voertuig bestaat met het opgegeven ID, retourneert de API een `404 Not Found`-respons.

```json
{
  "error": {
    "code": "not_found",
    "message": "No vehicle found with ID veh_01HX4K2QZ9P8V9."
  }
}
```

<Warning>
  Zorg ervoor dat u het volledige voertuig-ID inclusief het voorvoegsel `veh_` meestuurt in het pad. Een onvolledig of onjuist ID leidt altijd tot een `404`-fout.
</Warning>
