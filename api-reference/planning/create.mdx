title: "POST /planning — Planningentry aanmaken in Transportklok"
sidebarTitle: "Aanmaken"
description: "Wijs een transportopdracht toe aan een chauffeur en voertuig door een nieuwe planningentry aan te maken via de Transportklok REST API."
---

Met dit eindpunt maakt u een nieuwe planningentry aan door een transportopdracht te koppelen aan een chauffeur en voertuig. U geeft de geplande start- en eindtijd op voor de rit. Bij het aanmaken van een planningentry wordt automatisch de status van de gekoppelde transportopdracht bijgewerkt van `draft` naar `scheduled`. Als de opgegeven chauffeur of het opgegeven voertuig al is ingepland op het gewenste tijdstip, retourneert de API een conflictfout.

## Eindpunt

```
POST https://api.transportklok.nl/v1/planning
```

## Authenticatie

Voeg de volgende header toe aan elk verzoek:

```
Authorization: Bearer <api_key>
```

## Verzoekbody

<ParamField body="order_id" type="string" required>
  Het ID van de transportopdracht die u wilt inplannen (bijv. `ord_01HX4K2QZ9P8V7`). De opdracht moet de status `draft` hebben.
</ParamField>

<ParamField body="driver_id" type="string" required>
  Het ID van de chauffeur aan wie de rit wordt toegewezen (bijv. `drv_01HX4K2QZ9P8V8`).
</ParamField>

<ParamField body="vehicle_id" type="string" required>
  Het ID van het voertuig dat voor de rit wordt ingezet (bijv. `veh_01HX4K2QZ9P8V9`).
</ParamField>

<ParamField body="planned_start" type="string" required>
  Geplande starttijd van de rit in ISO 8601-formaat (bijv. `2024-06-15T07:30:00Z`).
</ParamField>

<ParamField body="planned_end" type="string" required>
  Geplande eindtijd van de rit in ISO 8601-formaat (bijv. `2024-06-15T16:00:00Z`).
</ParamField>

## Voorbeeldverzoek

```bash
curl -X POST "https://api.transportklok.nl/v1/planning" \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "order_id": "ord_01HX4K2QZ9P8V7",
    "driver_id": "drv_01HX4K2QZ9P8V8",
    "vehicle_id": "veh_01HX4K2QZ9P8V9",
    "planned_start": "2024-06-15T07:30:00Z",
    "planned_end": "2024-06-15T16:00:00Z"
  }'
```

## Voorbeeldrespons (201 Created)

```json
{
  "data": {
    "id": "pln_01HX4K2QZ9P8W1",
    "order_id": "ord_01HX4K2QZ9P8V7",
    "driver_id": "drv_01HX4K2QZ9P8V8",
    "vehicle_id": "veh_01HX4K2QZ9P8V9",
    "planned_start": "2024-06-15T07:30:00Z",
    "planned_end": "2024-06-15T16:00:00Z",
    "route": {
      "distance_km": 87.5,
      "estimated_duration_min": 95
    },
    "status": "confirmed",
    "created_at": "2024-06-12T14:30:00Z"
  }
}
```

## Responsvelden

<ResponseField name="data" type="object">
  Het nieuw aangemaakte planningentry-object.

  <Expandable title="Velden">
    <ResponseField name="data.id" type="string">
      Het unieke ID van de planningentry, voorafgegaan door `pln_`.
    </ResponseField>

    <ResponseField name="data.order_id" type="string">
      Het ID van de gekoppelde transportopdracht.
    </ResponseField>

    <ResponseField name="data.driver_id" type="string">
      Het ID van de toegewezen chauffeur.
    </ResponseField>

    <ResponseField name="data.vehicle_id" type="string">
      Het ID van het toegewezen voertuig.
    </ResponseField>

    <ResponseField name="data.planned_start" type="string">
      Geplande starttijd van de rit in ISO 8601-formaat.
    </ResponseField>

    <ResponseField name="data.planned_end" type="string">
      Geplande eindtijd van de rit in ISO 8601-formaat.
    </ResponseField>

    <ResponseField name="data.route" type="object">
      Routegegevens inclusief geschatte afstand en rijduur.
    </ResponseField>

    <ResponseField name="data.route.distance_km" type="number">
      Geschatte routeafstand in kilometers.
    </ResponseField>

    <ResponseField name="data.route.estimated_duration_min" type="integer">
      Geschatte rijtijd in minuten.
    </ResponseField>

    <ResponseField name="data.status" type="string">
      De planningsstatus: `draft`, `confirmed` of `completed`.
    </ResponseField>

    <ResponseField name="data.created_at" type="string">
      Tijdstip van aanmaken in ISO 8601-formaat.
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  Het aanmaken van een planningentry heeft een neveneffect: de status van de gekoppelde transportopdracht (het veld `order_id`) wordt automatisch bijgewerkt van `draft` naar `scheduled`. U hoeft dit niet afzonderlijk te doen via `PUT /orders/{id}`.
</Note>

## Foutrespons (409 Conflict)

Wanneer de opgegeven chauffeur of het opgegeven voertuig al is ingepland op het gewenste tijdstip:

```json
{
  "error": {
    "code": "CONFLICT",
    "message": "The requested time slot is unavailable.",
    "details": [
      {
        "resource": "driver",
        "id": "drv_01HX4K2QZ9P8V8",
        "message": "Driver 'drv_01HX4K2QZ9P8V8' is already scheduled between 2024-06-15T07:30:00Z and 2024-06-15T16:00:00Z (planning entry: pln_01HX4K2QZ9P8W0)."
      }
    ]
  }
}
```
