Met dit endpoint haalt u een gepagineerde lijst op van alle voertuigen in uw wagenpark. U kunt de resultaten filteren op voertuigtype en beschikbaarheidsstatus, zodat u snel inzicht heeft in welke voertuigen op een bepaald moment inzetbaar zijn voor transport.

## Endpoint

```
GET https://api.transportklok.nl/v1/vehicles
```

## Authenticatie

Voeg de volgende header toe aan elk verzoek:

```
Authorization: Bearer <api_key>
```

## Queryparameters

<ParamField query="status" type="string">
  Filter voertuigen op beschikbaarheidsstatus. Toegestane waarden: `available`, `in_use`, `maintenance`, `inactive`.
</ParamField>

<ParamField query="type" type="string">
  Filter voertuigen op type. Toegestane waarden: `van`, `truck`, `semi_trailer`.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Het paginanummer om op te halen. Begint bij `1`.
</ParamField>

<ParamField query="per_page" type="integer" default="25">
  Het aantal resultaten per pagina. Maximum: `100`.
</ParamField>

## Voorbeeldverzoek

```bash
curl --request GET \
  --url "https://api.transportklok.nl/v1/vehicles?status=available&type=truck&page=1&per_page=25" \
  --header "Authorization: Bearer <api_key>" \
  --header "Content-Type: application/json"
```

<Note>
  Het veld `apk_expiry` geeft de vervaldatum van de APK-keuring aan in `YYYY-MM-DD`-formaat. Houd bij het plannen van ritten altijd rekening met deze datum: voertuigen waarvan de APK is verlopen, mogen niet op de openbare weg rijden. Wij adviseren om voertuigen tijdig in te plannen voor keuring zodat uw wagenpark operationeel blijft.
</Note>

## Responsvelden

<ResponseField name="data" type="array">
  Een array van voertuigobjecten die voldoen aan de opgegeven filtercriteria.

  <Expandable title="Voertuigobject">
    <ResponseField name="id" type="string">
      De unieke identifier van het voertuig, voorafgegaan door `veh_`.
    </ResponseField>

    <ResponseField name="license_plate" type="string">
      Het kenteken van het voertuig in Nederlands formaat.
    </ResponseField>

    <ResponseField name="brand" type="string">
      Het merk van het voertuig.
    </ResponseField>

    <ResponseField name="model" type="string">
      Het model van het voertuig.
    </ResponseField>

    <ResponseField name="type" type="string">
      Het type voertuig. Mogelijke waarden: `van`, `truck`, `semi_trailer`.
    </ResponseField>

    <ResponseField name="payload_kg" type="integer">
      De maximale laadcapaciteit van het voertuig in kilogram.
    </ResponseField>

    <ResponseField name="volume_m3" type="integer">
      Het laadvolume van het voertuig in kubieke meter.
    </ResponseField>

    <ResponseField name="status" type="string">
      De huidige beschikbaarheidsstatus van het voertuig. Mogelijke waarden: `available`, `in_use`, `maintenance`, `inactive`.
    </ResponseField>

    <ResponseField name="apk_expiry" type="string">
      De vervaldatum van de APK-keuring in `YYYY-MM-DD`-formaat.
    </ResponseField>

    <ResponseField name="created_at" type="string">
      Tijdstip waarop het voertuig is geregistreerd, in ISO 8601-formaat.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  Metadata over de paginering van de resultatenset.

  <Expandable title="Meta-object">
    <ResponseField name="meta.current_page" type="integer">
      Het huidige paginanummer van de geretourneerde resultaten.
    </ResponseField>

    <ResponseField name="meta.per_page" type="integer">
      Het aantal resultaten per pagina zoals opgegeven in het verzoek.
    </ResponseField>

    <ResponseField name="meta.total" type="integer">
      Het totale aantal voertuigen dat voldoet aan de filtercriteria.
    </ResponseField>
  </Expandable>
</ResponseField>
