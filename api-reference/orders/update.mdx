Met dit eindpunt werkt u de gegevens van een bestaande transportopdracht bij. U hoeft alleen de velden mee te sturen die u wilt wijzigen; niet-opgegeven velden blijven ongewijzigd. Let op: u kunt uitsluitend opdrachten bewerken met de status `draft` of `scheduled`. Opdrachten die al onderweg zijn of zijn afgeleverd, kunnen niet meer worden aangepast via dit eindpunt.

## Eindpunt

```
PUT https://api.transportklok.nl/v1/orders/{id}
```

## Authenticatie

Voeg de volgende header toe aan elk verzoek:

```
Authorization: Bearer <api_key>
```

## Padparameter

<ParamField path="id" type="string" required>
  Het unieke ID van de transportopdracht die u wilt bijwerken (bijv. `ord_01HX4K2QZ9P8V7`).
</ParamField>

## Verzoekbody

Alle velden zijn optioneel. Stuur alleen de velden mee die u wilt wijzigen.

<ParamField body="reference" type="string" optional>
  Uw eigen referentienummer voor de opdracht.
</ParamField>

<ParamField body="pickup" type="object" optional>
  Bijgewerkte ophaalgegevens. Geef het volledige object mee; gedeeltelijke updates van het ophaalobject worden niet ondersteund.

  <Expandable title="Velden">
    <ParamField body="pickup.address" type="string" optional>
      Volledig ophaaladres inclusief postcode en stad.
    </ParamField>

    <ParamField body="pickup.contact_name" type="string" optional>
      Naam van de contactpersoon op de ophaallocatie.
    </ParamField>

    <ParamField body="pickup.contact_phone" type="string" optional>
      Telefoonnummer van de contactpersoon in internationaal formaat.
    </ParamField>

    <ParamField body="pickup.window_start" type="string" optional>
      Begintijd van het ophaaltijdvenster in ISO 8601-formaat.
    </ParamField>

    <ParamField body="pickup.window_end" type="string" optional>
      Eindtijd van het ophaaltijdvenster in ISO 8601-formaat.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="delivery" type="object" optional>
  Bijgewerkte aflevergegevens. Geef het volledige object mee; gedeeltelijke updates van het afleverobject worden niet ondersteund.

  <Expandable title="Velden">
    <ParamField body="delivery.address" type="string" optional>
      Volledig afleveradres inclusief postcode en stad.
    </ParamField>

    <ParamField body="delivery.contact_name" type="string" optional>
      Naam van de contactpersoon op de afleverlocatie.
    </ParamField>

    <ParamField body="delivery.contact_phone" type="string" optional>
      Telefoonnummer van de contactpersoon in internationaal formaat.
    </ParamField>

    <ParamField body="delivery.window_start" type="string" optional>
      Begintijd van het aflevertijdvenster in ISO 8601-formaat.
    </ParamField>

    <ParamField body="delivery.window_end" type="string" optional>
      Eindtijd van het aflevertijdvenster in ISO 8601-formaat.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="cargo" type="object" optional>
  Bijgewerkte ladingspecificaties.

  <Expandable title="Velden">
    <ParamField body="cargo.weight_kg" type="number" optional>
      Totaalgewicht van de lading in kilogram.
    </ParamField>

    <ParamField body="cargo.volume_m3" type="number" optional>
      Totaalvolume van de lading in kubieke meter.
    </ParamField>

    <ParamField body="cargo.type" type="string" optional>
      Type lading (bijv. `pallets`, `dozen`, `bulk`).
    </ParamField>

    <ParamField body="cargo.quantity" type="integer" optional>
      Aantal stuks, pallets of colli.
    </ParamField>

    <ParamField body="cargo.notes" type="string" optional>
      Aanvullende instructies of opmerkingen over de lading.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="driver_id" type="string" optional>
  Het ID van de nieuw toe te wijzen chauffeur.
</ParamField>

<ParamField body="vehicle_id" type="string" optional>
  Het ID van het nieuw toe te wijzen voertuig.
</ParamField>

## Voorbeeldverzoek

In dit voorbeeld werkt u het ladinggewicht bij en verschuift u het aflevertijdvenster:

```bash
curl -X PUT "https://api.transportklok.nl/v1/orders/ord_01HX4K2QZ9P8V7" \
  -H "Authorization: Bearer tk_live_xxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "cargo": {
      "weight_kg": 620,
      "volume_m3": 2.5,
      "type": "pallets",
      "quantity": 4,
      "notes": "Fragile - handle with care"
    },
    "delivery": {
      "address": "Keizersgracht 456, 1017 DW Amsterdam",
      "contact_name": "Sophie Jansen",
      "contact_phone": "+31687654321",
      "window_start": "2024-06-15T14:00:00Z",
      "window_end": "2024-06-15T16:00:00Z"
    }
  }'
```

<Warning>
  Opdrachten met de status `in_transit`, `delivered`, `invoiced` of `cancelled` kunnen **niet** worden bijgewerkt via dit eindpunt. Pogingen om dergelijke opdrachten te wijzigen resulteren in een `409 Conflict`-fout. Neem contact op met de ondersteuning indien u een foutief afgesloten opdracht moet corrigeren.
</Warning>

## Foutresponsen

**`401 Unauthorized`** — De API-sleutel ontbreekt of is ongeldig. Controleer de `Authorization`-header.

```json
{
  "error": "unauthorized",
  "message": "De opgegeven API-sleutel is ongeldig of verlopen.",
  "status": 401
}
```

**`400 Bad Request`** — De verzoekbody bevat ongeldige waarden voor één of meer velden.

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "The request body contains invalid or missing fields.",
    "details": [
      {
        "field": "cargo.weight_kg",
        "message": "Must be a positive number."
      }
    ]
  }
}
```

**`404 Not Found`** — Er bestaat geen opdracht met het opgegeven ID.

```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "No order found with ID 'ord_01HX4K2QZ9P8V7'."
  }
}
```

**`409 Conflict`** — Wanneer u probeert een reeds afgeleverde of anderszins afgesloten opdracht bij te werken.

```json
{
  "error": {
    "code": "CONFLICT",
    "message": "Order 'ord_01HX4K2QZ9P8V7' cannot be updated because its status is 'delivered'. Only orders with status 'draft' or 'scheduled' can be modified."
  }
}
```
