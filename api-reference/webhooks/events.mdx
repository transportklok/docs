Transportklok verstuurt webhook-events voor alle belangrijke acties en statuswijzigingen binnen het platform. Elk event bevat een gestructureerde payload met een uniek event-ID, het event-type, een tijdstempel en de bijbehorende gegevens. Hieronder vindt u een volledig overzicht van alle beschikbare event-typen, gegroepeerd per categorie, met voor elk event-type de volledige voorbeeldpayload.

<Info>
  Het veld `id` in elke webhook-payload is idempotent en uniek per event. U kunt dit ID veilig gebruiken voor deduplicatie in uw eigen systemen, zodat u herleveringen niet dubbel verwerkt.
</Info>

## Payloadstructuur

Elk webhook-event volgt dezelfde basisstructuur op het hoogste niveau:

| Veld         | Type   | Omschrijving                                                                    |
|--------------|--------|---------------------------------------------------------------------------------|
| `id`         | string | Unieke identifier van dit event, voorafgegaan door `evt_`. Gebruik dit voor deduplicatie bij herleveringen. |
| `event`      | string | Het event-type in de notatie `resource.action`, bijvoorbeeld `order.status_changed`. |
| `created_at` | string | Tijdstip waarop het event is aangemaakt, in ISO 8601-formaat.                   |
| `data`       | object | De event-specifieke payload. De inhoud verschilt per event-type (zie overzicht hieronder). |

## Transportopdrachten — `order.*`

<AccordionGroup>
  <Accordion title="order.created — Nieuwe opdracht aangemaakt">
    Wordt verstuurd zodra een nieuwe transportopdracht is aangemaakt in Transportklok.

    **Voorbeeldpayload:**

    ```json
    {
      "id": "evt_01HX4K2QZ9P8W5",
      "event": "order.created",
      "created_at": "2024-06-15T08:00:00Z",
      "data": {
        "order_id": "ord_01HX4K2QZ9P8V7",
        "reference": "KL-2024-00842"
      }
    }
    ```

    **Data-velden:**

    | Veld         | Type   | Omschrijving                                     |
    |--------------|--------|--------------------------------------------------|
    | `order_id`   | string | De unieke identifier van de nieuwe opdracht.     |
    | `reference`  | string | De externe referentie of klantreferentie van de opdracht. |
  </Accordion>

  <Accordion title="order.status_changed — Status gewijzigd">
    Wordt verstuurd wanneer de status van een transportopdracht wijzigt, bijvoorbeeld van `scheduled` naar `in_transit`.

    **Voorbeeldpayload:**

    ```json
    {
      "id": "evt_01HX4K2QZ9P8W6",
      "event": "order.status_changed",
      "created_at": "2024-06-15T14:32:00Z",
      "data": {
        "order_id": "ord_01HX4K2QZ9P8V7",
        "previous_status": "scheduled",
        "new_status": "in_transit"
      }
    }
    ```

    **Data-velden:**

    | Veld              | Type   | Omschrijving                                      |
    |-------------------|--------|---------------------------------------------------|
    | `order_id`        | string | De unieke identifier van de opdracht.             |
    | `previous_status` | string | De vorige status van de opdracht vóór de wijziging. |
    | `new_status`      | string | De nieuwe, huidige status van de opdracht.        |
  </Accordion>

  <Accordion title="order.updated — Opdrachtvelden gewijzigd">
    Wordt verstuurd wanneer één of meer velden van een bestaande opdracht zijn aangepast, zonder dat de status is gewijzigd.

    **Voorbeeldpayload:**

    ```json
    {
      "id": "evt_01HX4K2QZ9P8W7",
      "event": "order.updated",
      "created_at": "2024-06-15T10:15:00Z",
      "data": {
        "order_id": "ord_01HX4K2QZ9P8V7",
        "changed_fields": ["delivery_address", "contact_name"]
      }
    }
    ```

    **Data-velden:**

    | Veld             | Type   | Omschrijving                                                   |
    |------------------|--------|----------------------------------------------------------------|
    | `order_id`       | string | De unieke identifier van de opdracht.                         |
    | `changed_fields` | array  | Een lijst van veldnamen die zijn gewijzigd in deze update.    |
  </Accordion>

  <Accordion title="order.deleted — Opdracht verwijderd">
    Wordt verstuurd wanneer een transportopdracht definitief is verwijderd uit Transportklok.

    **Voorbeeldpayload:**

    ```json
    {
      "id": "evt_01HX4K2QZ9P8W8",
      "event": "order.deleted",
      "created_at": "2024-06-15T11:00:00Z",
      "data": {
        "order_id": "ord_01HX4K2QZ9P8V7"
      }
    }
    ```

    **Data-velden:**

    | Veld       | Type   | Omschrijving                              |
    |------------|--------|-------------------------------------------|
    | `order_id` | string | De unieke identifier van de verwijderde opdracht. |
  </Accordion>
</AccordionGroup>

## Planning — `planning.*`

<AccordionGroup>
  <Accordion title="planning.created — Planningentry aangemaakt">
    Wordt verstuurd wanneer een nieuwe planningentry is aangemaakt en een chauffeur en voertuig aan een opdracht zijn gekoppeld.

    **Voorbeeldpayload:**

    ```json
    {
      "id": "evt_01HX4K2QZ9P8WA",
      "event": "planning.created",
      "created_at": "2024-06-12T14:30:00Z",
      "data": {
        "planning_id": "pln_01HX4K2QZ9P8W1",
        "order_id": "ord_01HX4K2QZ9P8V7",
        "driver_id": "drv_01HX4K2QZ9P8V8"
      }
    }
    ```

    **Data-velden:**

    | Veld          | Type   | Omschrijving                                                  |
    |---------------|--------|---------------------------------------------------------------|
    | `planning_id` | string | De unieke identifier van de planningentry.                    |
    | `order_id`    | string | De identifier van de bijbehorende transportopdracht.          |
    | `driver_id`   | string | De identifier van de ingeplande chauffeur.                    |
  </Accordion>

  <Accordion title="planning.updated — Planningentry gewijzigd">
    Wordt verstuurd wanneer één of meer velden van een bestaande planningentry zijn aangepast, zoals een gewijzigde vertrektijd of een andere chauffeur.

    **Voorbeeldpayload:**

    ```json
    {
      "id": "evt_01HX4K2QZ9P8WB",
      "event": "planning.updated",
      "created_at": "2024-06-13T09:00:00Z",
      "data": {
        "planning_id": "pln_01HX4K2QZ9P8W1",
        "changed_fields": ["driver_id", "planned_start"]
      }
    }
    ```

    **Data-velden:**

    | Veld             | Type   | Omschrijving                                               |
    |------------------|--------|-------------------------------------------------------------|
    | `planning_id`    | string | De unieke identifier van de gewijzigde planningentry.      |
    | `changed_fields` | array  | Een lijst van veldnamen die zijn gewijzigd in deze update. |
  </Accordion>

  <Accordion title="planning.confirmed — Chauffeur heeft rit bevestigd">
    Wordt verstuurd wanneer een chauffeur de geplande rit heeft bevestigd via de Transportklok-app.

    **Voorbeeldpayload:**

    ```json
    {
      "id": "evt_01HX4K2QZ9P8WC",
      "event": "planning.confirmed",
      "created_at": "2024-06-14T07:45:00Z",
      "data": {
        "planning_id": "pln_01HX4K2QZ9P8W1",
        "driver_id": "drv_01HX4K2QZ9P8V8"
      }
    }
    ```

    **Data-velden:**

    | Veld          | Type   | Omschrijving                                       |
    |---------------|--------|-----------------------------------------------------|
    | `planning_id` | string | De unieke identifier van de planningentry.         |
    | `driver_id`   | string | De identifier van de chauffeur die heeft bevestigd. |
  </Accordion>

  <Accordion title="planning.completed — Rit afgerond">
    Wordt verstuurd wanneer een planningentry de status `completed` bereikt en de rit als afgerond is gemarkeerd.

    **Voorbeeldpayload:**

    ```json
    {
      "id": "evt_01HX4K2QZ9P8WD",
      "event": "planning.completed",
      "created_at": "2024-06-15T16:05:00Z",
      "data": {
        "planning_id": "pln_01HX4K2QZ9P8W1",
        "order_id": "ord_01HX4K2QZ9P8V7",
        "driver_id": "drv_01HX4K2QZ9P8V8"
      }
    }
    ```

    **Data-velden:**

    | Veld          | Type   | Omschrijving                                              |
    |---------------|--------|-----------------------------------------------------------|
    | `planning_id` | string | De unieke identifier van de afgeronde planningentry.      |
    | `order_id`    | string | De identifier van de bijbehorende transportopdracht.      |
    | `driver_id`   | string | De identifier van de chauffeur die de rit heeft afgerond. |
  </Accordion>
</AccordionGroup>

## Voertuigen — `vehicle.*`

<AccordionGroup>
  <Accordion title="vehicle.created — Nieuw voertuig geregistreerd">
    Wordt verstuurd wanneer een nieuw voertuig is toegevoegd aan de wagenparkregistratie in Transportklok.

    **Voorbeeldpayload:**

    ```json
    {
      "id": "evt_01HX4K2QZ9P8WH",
      "event": "vehicle.created",
      "created_at": "2024-01-10T08:00:00Z",
      "data": {
        "vehicle_id": "veh_01HX4K2QZ9P8V9",
        "license_plate": "12-AB-34",
        "type": "truck"
      }
    }
    ```

    **Data-velden:**

    | Veld            | Type   | Omschrijving                                        |
    |-----------------|--------|-----------------------------------------------------|
    | `vehicle_id`    | string | De unieke identifier van het nieuwe voertuig.       |
    | `license_plate` | string | Het kenteken van het voertuig in Nederlands formaat. |
    | `type`          | string | Het voertuigtype. Mogelijke waarden: `van`, `truck`, `semi_trailer`. |
  </Accordion>

    **Data-velden:**

    | Veld             | Type   | Omschrijving                                                 |
    |------------------|--------|--------------------------------------------------------------|
    | `vehicle_id`     | string | De unieke identifier van het bijgewerkte voertuig.           |
    | `changed_fields` | array  | Een lijst van veldnamen die zijn gewijzigd in deze update.   |
  </Accordion>

  </Accordion>
