De Transportklok REST API stelt u in staat om uw eigen systemen, workflows en applicaties direct te koppelen aan het Transportklok platform. Alle functionaliteit die u kent uit de webinterface — van het aanmaken van transportopdrachten tot het synchroniseren van planningen — is beschikbaar via gestandaardiseerde HTTP-verzoeken met JSON als gegevensformaat. De API is ontworpen volgens de REST-architectuurprincipes en werkt met elke programmeertaal of tool die HTTP ondersteunt.

## Base URL

Alle API-verzoeken worden verstuurd naar de volgende basis-URL:

```
https://api.transportklok.nl/v1
```

## OpenAPI-specificatie

De volledige machineleesbare API-specificatie in OpenAPI 3.0-formaat is beschikbaar op:

```
https://api.transportklok.nl/v1/openapi.json
```

U kunt dit bestand importeren in **Swagger UI** of andere OpenAPI-compatibele tools (zoals Redoc, Postman of Insomnia) om de volledige API interactief te verkennen, verzoeken te testen en automatisch client-code te genereren. Laad de URL rechtstreeks in Swagger UI via `File → Import URL` om direct aan de slag te gaan.

<Note>
  De OpenAPI-specificatie wordt bijgewerkt bij elke nieuwe API-versie. Neem contact op met ondersteuning voor een overzicht van wijzigingen.
</Note>

## Aan de slag met de API

<Steps>
  <Step title="API-sleutel aanmaken">
    Voordat u verzoeken kunt versturen, heeft u een geldige API-sleutel nodig. Maak een sleutel aan via **Instellingen → Integraties → API-sleutels** in uw Transportklok-account. Zie de [authenticatiedocumentatie](/api-reference/authentication) voor een stap-voor-stap uitleg.
  </Step>
