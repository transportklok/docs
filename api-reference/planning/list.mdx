Met dit eindpunt haalt u een overzicht op van alle planningsentries binnen een opgegeven datumbereik. U kunt de resultaten verder filteren op chauffeur, voertuig of planningsstatus. Dit eindpunt is met name geschikt voor het ophalen van de dagplanning of weekplanning, zodat u een volledig beeld heeft van de ingeplande ritten en capaciteitsbenutting.

## Eindpunt

```
GET https://api.transportklok.nl/v1/planning
```

## Authenticatie

Voeg de volgende header toe aan elk verzoek:

```
Authorization: Bearer <api_key>
```

## Queryparameters

<ParamField query="date_from" type="string" required>
  Begindatum van het op te halen planningsbereik, in ISO 8601-formaat (bijv. `2024-06-15T00:00:00Z`). Dit veld is verplicht.
</ParamField>

<ParamField query="date_to" type="string" required>
  Einddatum van het op te halen planningsbereik, in ISO 8601-formaat (bijv. `2024-06-15T23:59:59Z`). Dit veld is verplicht.
</ParamField>

<ParamField query="driver_id" type="string" optional>
  Filter planningsentries op het ID van de toegewezen chauffeur (bijv. `drv_01HX4K2QZ9P8V8`).
</ParamField>

<ParamField query="vehicle_id" type="string" optional>
  Filter planningsentries op het ID van het toegewezen voertuig (bijv. `veh_01HX4K2QZ9P8V9`).
</ParamField>

<ParamField query="status" type="string" optional>
  Filter op de planningsstatus. Mogelijke waarden: `draft`, `confirmed`, `completed`.
</ParamField>

<ParamField query="page" type="integer" optional default="1">
  Het paginanummer van de resultatenset.
</ParamField>

<ParamField query="per_page" type="integer" optional default="25">
  Het aantal resultaten per pagina. Maximaal `100`.
</ParamField>

## Voorbeeldverzoek

```bash
curl -X GET "https://api.transportklok.nl/v1/planning?date_from=2024-06-15T00:00:00Z&date_to=2024-06-15T23:59:59Z&status=confirmed" \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json"
```

## Responsvelden

<ResponseField name="data" type="array">
  Een array van planningsentries die voldoen aan de opgegeven filtercriteria.

  <Expandable title="Velden per planningentry">
    <ResponseField name="data[].id" type="string">
      Het unieke ID van de planningentry (bijv. `pln_01HX4K2QZ9P8W1`).
    </ResponseField>

    <ResponseField name="data[].order_id" type="string">
      Het ID van de gekoppelde transportopdracht.
    </ResponseField>

    <ResponseField name="data[].driver_id" type="string">
      Het ID van de toegewezen chauffeur.
    </ResponseField>

    <ResponseField name="data[].vehicle_id" type="string">
      Het ID van het toegewezen voertuig.
    </ResponseField>

    <ResponseField name="data[].planned_start" type="string">
      Geplande starttijd van de rit in ISO 8601-formaat.
    </ResponseField>

    <ResponseField name="data[].planned_end" type="string">
      Geplande eindtijd van de rit in ISO 8601-formaat.
    </ResponseField>

    <ResponseField name="data[].route" type="object">
      Routegegevens inclusief geschatte afstand en rijduur.
    </ResponseField>

    <ResponseField name="data[].route.distance_km" type="number">
      Geschatte routeafstand in kilometers.
    </ResponseField>

    <ResponseField name="data[].route.estimated_duration_min" type="integer">
      Geschatte rijtijd in minuten.
    </ResponseField>

    <ResponseField name="data[].status" type="string">
      De planningsstatus: `draft`, `confirmed` of `completed`.
    </ResponseField>

    <ResponseField name="data[].created_at" type="string">
      Tijdstip van aanmaken in ISO 8601-formaat.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object">
  Pagineringsinformatie over de volledige resultatenset.

  <Expandable title="Velden">
    <ResponseField name="meta.total" type="integer">
      Totaal aantal planningsentries dat overeenkomt met de filtercriteria.
    </ResponseField>

    <ResponseField name="meta.page" type="integer">
      Het huidige paginanummer.
    </ResponseField>

    <ResponseField name="meta.per_page" type="integer">
      Het aantal resultaten per pagina.
    </ResponseField>
  </Expandable>
</ResponseField>
